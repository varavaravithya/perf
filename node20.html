ตัวอย่าง:

การตอบสนองของคอมพิวเตอร์ Gateway มักแสดงโดยเวลาการตอบสนอง-คือ
  1. Response Time: เวลาระหว่างเพ็คเกตเข้ามาถึง Gateway และส่งผ่านออกไป
  2. Throughput: อัตราความสำเร็จของ Gateway วัดเป็นจำนวนเพ็คเกตที่สามารถส่งได้ในหน่วยเวลาเรียกว่าอัตราการส่งผ่าน
  3. Utilization: ภาระงานเป็นการวัดเปอร์เซ็นต์ของเวลาที่ Gateway ทำงานอยู่เทียบกับเวลาเฝ้าสังเกตทั้งหมด
อุปกรณ์ที่มีภาระงานมากที่สุด (Highest Utilization) ถือว่าเป็นอุปกรณ์ที่เป็นคอขวด (Bottleneck) การปรับปรุงประสิทธิภาพของระบบจะได้ประสิทธิภาพเพิ่มขึ้นสูงสุด ถ้าเราปรับปรุงอุปกรณ์ที่เป็นคอขวด ดังนั้นการหาภาระงานของอุปกรณ์ใดๆ ในระบบ จึงเป็นการประเมินประสิทธิภาพที่สำคัญ ในกรณีที่ระบบตอบสนองผิดพลาด ผลการตอบสนองที่ผิดจะส่งผลต่อการทำงานโดยตรง จึงจำเป็นต้องแจกแจงความผิดพลาดที่อาจเกิดขึ้น การแสดงตัววัดประสิทธิภาพในกรณีที่แสดงผลตอบสนองผิดพลาดจะแสดงเป็นความน่าจะเป็นของความผิดพลาดนั้นๆ ตัวอย่างเช่น ในคอมพิวเตอร์ Gateway เราต้องการทราบความน่าจะเป็นที่จะเกิดความผิดพลาด หนึ่งบิต สองบิต ... ในขณะที่ทำการส่งผ่านเพ็คเกต หรือเราอยากทราบความน่าจะเป็นที่เพ็คเกตไม่สมบูรณ์ ในกรณีที่ระบบไม่ตอบสนอง อาจเกิดจากระบบไม่ทำงานเสียหาย หรือไม่ได้บริการ จึงมีความจำเป็นต้องแสดงตัววัดประสิทธิภาพในกรณีนี้ ตัวอย่างเช่น คอมพิวเตอร์ Gateway อาจสามารถใช้งานได้ 99.99 เปอร์เซ็นต์ของเวลาทั้งหมด ตัววัดประสิทธิภาพที่อิงกับผลการตอบสนองของระบบ 3 ชนิด ได้แก่ สำเร็จ, ผิดพลาด, และไม่บริการนั้น เรียกว่า ความเร็ว (Speed), ความน่าเชื่อถือ (Reliability), และความสามารถในการให้บริการ (Availability) จะเห็นได้อย่างชัดเจนว่าในแต่ละบริการที่อยู่ในระบบจะมีตัววัดทั้ง 3 ตัว เข้าเกี่ยวข้องด้วยเสมอ ในส่วนของจำนวนตัววัดในการประเมินประสิทธิภาพ ระบบที่มีบริการมากกว่าบริการหนึ่งตัววัดประสิทธิภาพจะเพิ่มขึ้นเป็นอัตราส่วน สำหรับตัววัดประสิทธิภาพหลายตัวการหาค่าเฉลี่ยเพื่อแสดงผลมีความสำคัญ อย่างไรก็ตามต้องพิจารณาความแปรปรวนประกอบด้วย ตัวอย่างเช่น ถ้าเวลาเฉลี่ยในการตอบสนองของระบบ มีค่าสูง และความแปรปรวนมีค่าสูง ซึ่งทั้ง 2 ตัววัด จะส่งผลทำให้ระบบไม่สามารถทำงานได้อย่างมีประสิทธิภาพ ในกรณีนี้เราจำเป็นต้องศึกษาตัววัดทั้ง 2 ตัว
